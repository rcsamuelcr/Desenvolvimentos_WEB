Empty Values em JavaScript

Em JavaScript, um "empty value" representa a ausência de um valor útil ou significativo. Esses valores são importantes para verificar se uma variável está vazia, indefinida ou nula.

Principais Empty Values em JavaScript

  • "" — string vazia
  • 0 — número zero
  • null — valor intencionalmente vazio
  • undefined — valor não atribuído
  • false — valor booleano falso
  • NaN — "Not a Number"
  • [] — array vazio
  • {} — objeto vazio

Exemplo de verificação de Empty Values

Você pode usar o operador ! para verificar se um valor é "falsy" (considerado vazio ou falso): 


// Exemplo de diferentes valores vazios
const valores = ["", 0, null, undefined, false, NaN];

valores.forEach(valor => {
  if (!valor) {
    console.log(`Valor vazio detectado:`, valor);
  }
});

Not a Number

Em JavaScript, um valor é considerado tipo de dado que representa um valor numérico indefinido ou não representável. Geralmente um NaN surge de:

  • Conversões de números com falha:
    Tentativa de converter uma string que não representa um número válido usando funções como parseInt()ou parseFloat().
    • console.log(parseInt("hello")); // NaN
  • Operações matemáticas inválidas: Operações que não produzem um número real como:
    • Dividir zero por zero;
    • Tirar a raiz quadrada de um número negativo;
    • Realizar operações consigo NaN mesmo.
    
    console.log(0 / 0); // NaN
    console.log(Math.sqrt(-1)); // NaN
    console.log(NaN + 5); // NaN

Comparação NaN única porque não é igual a si mesma, mesmo quando se usa igualdade estrita (===).

console.log(NaN === NaN); // false

Dica importante

Embora null e undefined sejam ambos "valores vazios", eles têm significados diferentes:

  • null → valor intencionalmente vazio (setado pelo programador)
  • undefined → valor ainda não definido (ex: variável não inicializada)

let nome;
console.log(nome); // undefined

let idade = null;
console.log(idade); // null
